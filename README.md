System Design: General Guide
============================

System design interviews and real-world design sessions share the same underlying structure.
The goal is never to ~~produce the "correct" answer~~ - it's to demonstrate ***structured thinking***, sound ***trade-off reasoning***, and the ability to communicate ***architectural decisions*** clearly.

This guide is a reusable procedure.
Every section follows the same pattern:

- ***What***: **what** the step is and **why** it matters
- ***How***: the **questions** to ask and the **tools** to use
- Example: the step applied to a worked example
